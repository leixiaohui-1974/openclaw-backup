# CHS多角色评审系统

对指定章节执行多角色评审。

## 输入

用户提供待评审文件路径。

## 评审流程

### 1. 读取上下文
- 读取 `knowledge-base/experience/common-errors.md` 常见错误清单
- 读取 `knowledge-base/experience/reviewer-lessons.md` 评审经验
- 读取待评审章节全文
- 确认书目类型（教材/专著/先导版），选择对应质量标准

### 2. 并行执行评审角色

**Role A 学科教师**（可教性视角）:
- 覆盖度、难度梯度、例题质量、习题质量、可教性
- 输出: X/10 + 修改建议

**Role B 学科专家**（学术准确性视角）:
- 准确性、文献覆盖、前沿性、原创性
- 自引率检查：T1=10-15%, T2=7-10%, M=5-8%
- 输出: X/10 + 修改建议

**Role C 行业工程师**（工程实用性视角）:
- 工程真实性、可落地性、可读性
- 输出: X/10 + 修改建议

**Role D 国际读者**（仅英文版）:
- 术语准确性、国际通用性、英文质量
- 输出: X/10 + 修改建议

### 3. 汇总
- 按严重性分级: 🔴致命 → 🟡重要 → 🟢建议
- 给出综合评分和是否通过的判定
- 列出需修改的具体条目

### 4. 可选：自动修改
如用户同意，直接执行 🔴 和 🟡 级修改，然后重新运行质量检查清单。
