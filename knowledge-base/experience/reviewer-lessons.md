# 审稿经验教训

> reviewer Agent 专用。从历次审稿中总结的经验。

## 审稿流程教训

1. **必须先跑三件套再审**：formula_audit.py + sympy_verify.py + citation_verify.py。不跑工具就说"全部通过"是最大的错误。
2. **不要因为"差不多"就放过**：公式差一个下标就是完全不同的物理量。
3. **逐条检查不要跳**：检查了5个就说"其余没问题"是禁止行为。

## 高频遗漏

4. **行内公式容易漏**：不只检查独立公式行，`$...$` 行内公式也要查。
5. **参考文献格式不一致**：有的用 (Author, Year)，有的用 [N]，必须统一。
6. **图表编号断裂**：删除某张图后后续编号没更新。

## 评分标准经验

- A 级（可发布）：P0=0, P1≤2, P2≤5
- B 级（小修后可发布）：P0=0, P1≤5
- C 级（需大修）：P0≤2 或 P1>5
- D 级（需重写）：P0>2

## 验证率基准

- 引用验证率 ≥ 90% 才可通过
- 团队论文 (T01-T43) 必须 100% 匹配 verified-refs.md
- 标准文献 (GB/T, ISO) 编号必须完全正确
