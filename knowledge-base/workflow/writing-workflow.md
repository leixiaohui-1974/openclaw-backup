# 写作工作流：要点先行法 (Bullet-Point-First Method)

> 研究表明：先写要点大纲再让AI生成散文，质量远超直接生成全文。
> 原因：学术写作需要A→B→C→结论的逻辑链，AI直接写容易跳步。

## 强制流程

### 第1步：结构化要点（人类或AI辅助）

收到写作任务后，**先输出要点大纲**，不直接写正文。格式：

```
## X.Y 节标题

### 本节目标
- 要传达的核心论点（1句话）
- 在全章中的角色（承上启下/引入新概念/证明定理/...）

### 逻辑链
1. [背景] 从上节的XX结论出发...
2. [问题] 现有方法的局限是...
3. [方法] 提出的解决方案是...（关键公式：eq.X.Y from master-formulas.md）
4. [验证] 用XX案例验证，结果是...
5. [意义] 这意味着...，为下节XX做铺垫

### 需要引用
- [REF] Litrico & Fromion 2004 → IDZ模型原始出处
- [REF] Lei et al. 2019 → 我们的改进工作
- [REF-TODO] 需要找一篇关于XX的最新文献

### 需要图表
- [FIG] 控制框图，显示MPC与分布式协调的关系
- [FIG-TODO] 需要实验数据对比图
```

### 第2步：用户确认或自动确认

- 如果用户在场：输出要点等确认后再写
- 如果是流水线自动执行：main检查要点是否覆盖知识库相关内容，自动确认

### 第3步：基于要点生成散文

读取要点大纲 + style-blueprint → 逐节生成，每节完成后检查：
- ✅ 逻辑链完整（要点中的每个步骤都体现在正文中）
- ✅ 公式衔接（前有引入，后有解释）
- ✅ 引用具体（不是"文献[1]研究了"而是"Litrico提出了"）
- ✅ 段落≤300字

### 第4步：自检清单

写完后对照检查：
- [ ] 每段有论点句开头？
- [ ] 公式前后有文字衔接？
- [ ] 术语与chs-terms.md一致？
- [ ] 图表说明具体（有数值）？
- [ ] 无禁止风格（见style-blueprint）？

## 特殊场景

### 写论文Introduction
要点必须包含：
1. 研究领域的重要性（2-3句，不用"随着...的发展"）
2. 现有研究的3-5个主要方向（每个方向1-2个代表性工作）
3. 研究空白（gap）的明确陈述
4. 本文贡献的具体列表（3-4条，每条一句话）
5. 论文结构概述

### 写教材章节
要点必须包含：
1. 本章在全书中的位置（前置知识→本章→后续应用）
2. 学习目标（3-5条）
3. 核心概念清单（每个概念：定义+物理意义+数学表达）
4. 例题/案例设计思路
5. 本章小结的关键takeaway
